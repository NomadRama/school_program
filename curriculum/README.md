# Curriculum — источник правды учебной программы ShokanLab

Эта папка содержит **всю учебную программу школы** в виде текстовых файлов. Git + GitHub — единственное хранилище. Notion не используется.

## Зачем так

- **История.** `git log` показывает все изменения, кто и когда правил
- **Откат.** Любая правка отменяется одной командой
- **Нет зависимости** от внешних сервисов
- **Переносимо** — работает на любой машине, любой ОС
- **Открыто** — будущие менторы клонируют репо и помогают через Pull Request

## Формат файлов

| Файл | Формат | Кто редактирует | Для чего |
|---|---|---|---|
| `content.md` | Markdown | Рама | Текст методички, который видит ученик |
| `spec.yaml` | YAML | Рама | Метаданные: длительность, компетенции, железо |
| `prompt.md` | Markdown | Рама | Мастер-промпт для ИИ, копируется учеником в чат |
| `mentor.md` | Markdown | Рама | Критерий зачёта, наблюдения — ученику не показываем |
| `code/*.py` | Python | Рама | Референсный код, на который опирается методичка |
| `diagrams/*.svg` | SVG | Рама | Иллюстрации (блок-диаграммы, схемы, визуалы) |

## Принципы

1. **Один файл — одна ответственность.** Текст в `content.md`, визуал в `diagrams/*.svg`, данные в `spec.yaml`. Не смешиваем.
2. **Не дублируем.** Если факт уже в `spec.yaml` (например, длительность) — в `content.md` ссылаемся, а не копируем.
3. **Понятные имена.** `m1-controllable-lamp` лучше чем `module_1` — сразу ясно о чём.
4. **Не сохраняем бинарники** (фото/видео) в этом репо. Фото работ — в Google Drive со ссылками в `content.md`.
5. **Приватные данные учеников** НЕ попадают сюда. Для учеников и портфолио — отдельная база (Airtable).

## Сайт

Сайт школы (`/docs/`) собирается из этой папки. Правка идёт в `curriculum/`, потом генерируется HTML. Прямые правки в `/docs/` допустимы для экспериментов, но всегда переносятся обратно в `curriculum/` — иначе уйдут при следующей сборке.

## Как вносить изменения

```bash
# 1. Правите файл в curriculum/
# 2. Коммит с осмысленным сообщением
git add curriculum/stage-1/m1-controllable-lamp/content.md
git commit -m "m1: упростил объяснение полярности LED"
git push

# 3. Если нужно — пересобрать сайт (пока вручную)
```

## Что НЕ хранится здесь

- Личные данные учеников (имена, телефоны, оценки) → Airtable
- Фото/видео работ → Google Drive
- Финансы и расписание → Google Sheets + Calendar
- Паспорта компонентов железа → пока нигде, будет отдельная папка `hardware/` позже

## Версии

Семантика для modules:
- `v0.x` — черновик, в работе
- `v1.0` — прошёл пилот, готов к занятиям
- `v1.x` — доработки после реальных учеников

Текущая версия Модуля 1 записана в его `spec.yaml` (`version`).
